3 Struct representing an open PDF document in the reader.
4
5 Holds the full PDF binary, the merged cross-reference table, the most-recent
6 trailer dictionary, a lazy object-resolution cache, and memoized page refs.
7
8 The struct is immutable — every `Pdf.Reader.*` function that resolves objects
9 returns an updated copy with a warmer cache. Dropping the updated copy is safe
10 (correctness is preserved) but re-resolving the same object will incur a
11 re-parse. Thread the returned doc forward for performance.
12
13 Callers do not construct `Document` directly; obtain one via `Pdf.Reader.open/1`.
14
15 ## Cache key conventions
16
17 The `:cache` field is a plain `%{}` map. Keys used by reader subsystems:
18
19 - `{n, g}` — resolved object with xref ref `(n, g)` (used by `ObjectResolver`)
20 - `{:font_decoder, {n, g}}` — cached decoder closure for the font at ref `(n, g)`,
21 built by `Pdf.Reader.Font.build_decoder/2`. Present only for fonts accessed via
22 an indirect reference. Inline font dicts (embedded literally in a resources dict)
23 are NOT cached — they are rebuilt on every call.
24 - `{:page_resources, {n, g}}` — resolved `/Resources` map for the leaf page at
25 ref `(n, g)`. Written by `Pdf.Reader.resolve_page_resources/4` after the first
26 `/Parent`-chain walk for a given page. Subsequent calls for the same page ref
27 short-circuit the walk and return the cached map directly. Intermediate ancestor
28 nodes are NOT cached — only the leaf page ref is used as the key.
29
30 ## Recovery mode
31
32 When opened with `recover: true`, the struct carries two additional fields:
33
34 - `:recover_mode` — `true` when recovery is active; `false` (default) for strict mode.
35 - `:recovery_log` — a reverse-prepend accumulator of structured recovery event tuples.
36 Exposed in chronological (oldest-first) order via `Pdf.Reader.recovery_log/1`.
37
38 Closed set of recovery event tuples (PDF 1.7 § 7.5, § 7.5.4, § 7.5.5, § 7.5.8):
39
40 | Tuple | Meaning |
41 |---|---|
42 | `{:xref_recovered, n}` | Linear scan recovered `n` object entries (§ 7.5.4, § 7.5.8) |
43 | `{:eof_marker_missing, :linear_scan_used}` | `%%EOF` absent; linear scan was invoked (§ 7.5.5) |
44 | `{:page_failed, page_n_or_ref, reason}` | A page was skipped. `page_n_or_ref` is either a `non_neg_integer()` page index OR a `{n, g}` ref-key tuple (used when iteration happens by `/Kids` ref before pages are indexed); `reason` is an atom or term |
45 | `{:font_skipped, page_n, font_name, reason}` | Font replaced with U+FFFD fallback |
46 | `{:page_tree_recovered, n}` | Catalog/Pages fallback found `n` page objects |
