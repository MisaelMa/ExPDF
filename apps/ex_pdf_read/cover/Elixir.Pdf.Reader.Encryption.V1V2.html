19
20 ## Algorithm 2 — File Encryption Key (PDF 1.7 § 7.6.3.3, step a–h)
21
22 1. Pad the user password to 32 bytes via `PasswordPad.pad/1`.
23 2. MD5 streaming: hash(padded_pw ++ /O ++ <<P::little-32>> ++ /ID[0]).
24 3. If R ≥ 4 and `encrypt_metadata == false`, append `<<0xFF,0xFF,0xFF,0xFF>>`.
25 4. If R ≥ 3, iterate MD5 × 50 on the first `key_len` bytes of the digest.
26 5. Truncate to first `key_len = Length / 8` bytes (min 5 for V1).
27
28 ## Algorithm 4 — User Auth for R=2
29
30 1. Derive file key via Algorithm 2.
31 2. RC4-encrypt the 32-byte padding constant with the file key.
32 3. Compare result byte-for-byte with /U (32 bytes).
33
34 ## Algorithm 5 — User Auth for R≥3
35
36 1. Derive file key via Algorithm 2.
37 2. MD5(padding_constant ++ /ID[0]) → 16 bytes.
38 3. RC4-encrypt those 16 bytes with the file key.
39 4. For i in 1..19: XOR each byte of file_key with i → RC4-encrypt previous result.
40 5. Compare final 16 bytes to the first 16 bytes of /U.
41
42 ## Algorithm 7 — Owner → User Password
43
44 1. Pad the owner password to 32 bytes.
45 2. MD5 it; if R ≥ 3, iterate MD5 × 50 on the full 16 bytes.
46 3. Truncate to `key_len` bytes → this becomes the RC4 key.
47 4. For R=2: RC4-decrypt /O once.
48 5. For R≥3: 20 iterative RC4 passes in reverse order (i=19 down to 0),
49 XORing each key byte with i.
50 6. The result is the padded user password. Feed it to `authenticate_user/2`.
