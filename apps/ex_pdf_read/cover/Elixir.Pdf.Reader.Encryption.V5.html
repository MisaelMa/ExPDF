18 ## Algorithm 2.B — PDF 2.0 iterative SHA mixing
19
20 Implements ISO 32000-2 § 7.6.4.3.4 "Algorithm 2.B" (also called
21 `calculatePDF20Hash` in Mozilla pdf.js).
22
23 ```
24 K = SHA-256(initial_data)
25 round = 0
26 repeat while round < 64 OR last byte of E > (round - 32):
27 K1 = (password ++ K ++ user_bytes) × 64
28 E = AES-128-CBC-encrypt(K1, key=K[0..15], IV=K[16..31], no padding)
29 sum = sum of first 16 bytes of E (mod 3)
30 K = SHA-256(E) if sum==0
31 SHA-384(E) if sum==1
32 SHA-512(E) if sum==2
33 round += 1
34 return K[0..31]
35 ```
36
37 Where:
38 - `initial_data` = password ++ salt (++ user_bytes for owner path)
39 - `user_bytes` = empty binary for user path, U[0..47] for owner path
40
41 ## Algorithm 8 — User Password Authentication (V5/R6)
42
43 1. Truncate password to 127 bytes (UTF-8 encoded).
44 2. hash = `pdf20_hash(password, password ++ U[32..39], <<>>)`
45 3. If hash == U[0..31] → authentication passes.
46 4. Compute `ue_key = pdf20_hash(password, password ++ U[40..47], <<>>)`
47 5. AES-256-CBC-decrypt `/UE` with `ue_key` and IV = 16 zero bytes.
48 6. Return `{:ok, file_key}` (32 bytes).
49
50 ## Algorithm 9 — Owner Password Authentication (V5/R6)
51
52 1. Truncate password to 127 bytes.
53 2. U = handler.u (full 48 bytes).
54 3. hash = `pdf20_hash(password, password ++ O[32..39] ++ U, U)`
55 4. If hash == O[0..31] → authentication passes.
56 5. Compute `oe_key = pdf20_hash(password, password ++ O[40..47] ++ U, U)`
57 6. AES-256-CBC-decrypt `/OE` with `oe_key` and IV = 16 zero bytes.
58 7. Return `{:ok, file_key}` (32 bytes).
59
60 ## Algorithm 10 — File Key Recovery
61
62 Embedded in `authenticate_user/2` and `authenticate_owner/2`. After
63 successful hash comparison, the appropriate key-derivation hash is computed
64 and AES-256-CBC decryption (no padding, IV = 16 zero bytes) of `/UE` or
65 `/OE` yields the 32-byte file encryption key.
66
67 ## V5 decryption (streams and strings)
68
69 For V5, the file encryption key is used DIRECTLY — no per-object key
70 derivation step (unlike V1/V2/V4 which use `ObjectKey.derive/4`). This is
71 per PDF 2.0 § 7.6.5 (R-ENC26).
72
73 Format: first 16 bytes of ciphertext = IV; remainder = AES-256-CBC ciphertext.
74 After decryption, PKCS7 padding is stripped manually (last byte `N`, validate
75 `1 ≤ N ≤ 16`, strip `N` bytes). Invalid padding returns `:error` without
76 raising.
77
78 ## PKCS7 unpadding (shared helper)
79
80 The same unpad logic is used by V4 (AES-128-CBC) and V5 (AES-256-CBC).
81 Rather than depending on V4 (creating a cross-module coupling), V5 contains
82 its own private implementation. The design decision is documented here: if
83 a shared `Pdf.Reader.Encryption.AES` helper module is introduced in a future
84 phase, both V4 and V5 can be refactored to delegate to it without a breaking
85 change.
